Write a Node.js CLI tool called `wordcount` (single file `wordcount.js`) that:

- Accepts a file path as the first argument (`node wordcount.js <path>`)
- Prints a one-line summary to stdout: word count, line count, character count, and the longest line length, separated by tabs
- Handles file-not-found errors gracefully (prints an error to stderr and exits with code 1)
- Uses only the Node standard library (no external deps)

Also produce:
- `wordcount.test.js` — a small Node-built-in test (using `node:test` and `node:assert`) covering: a normal multi-line file case, an empty-file case, and a missing-file case
- `README.md` — short, with install + usage + sample output

Output every file with a `=== FILE: <path> ===` header followed by its full content. Single deliverable, ready to drop into a folder and run.
